# 品牌诊断 Skill

## 1. Skill 定位

该 Skill 用于完成品牌全案中的「品牌诊断层」，生成更接近咨询公司正式提案报告的中文品牌诊断 HTML 报告。

它帮助使用者从品牌当前处境、行业与入口变化、竞品心智格局、用户心智与场景洞察、品牌资产与断点、核心矛盾与战略机会、品牌升级方向、下一步验证建议等维度，判断品牌真正的问题、机会、风险和后续战略输入。

该 Skill 不是信息整理工具，也不是摘要生成器，而是一个可复用的咨询式品牌诊断 HTML 报告生成器。输出必须结论先行、问题导向、证据支撑、分析推导、战略启示明确，并适合在浏览器打开或继续导出 PDF。

## 2. 适合使用人群

- 品牌营销负责人
- 高级品牌经理
- 品牌策略顾问
- 增长营销负责人
- 内容营销负责人
- IP 商业化负责人
- 新业务品牌负责人
- 年度品牌规划负责人
- 创始人或业务负责人

## 3. 适合使用场景

- 品牌全案前期诊断
- 年度品牌规划前
- 品牌升级或重塑前
- 新业务品牌定位前
- Campaign 策略制定前
- 内容矩阵规划前
- 渠道与入口策略规划前
- 用户增长策略制定前
- IP 商业化规划前
- 品牌现状复盘
- 竞品压力较大时
- 品牌增长乏力时

## 4. 核心升级点

新版 Skill 增加以下能力：

1. 诊断依据页：每个核心结论必须标注依据类型、依据说明、可信度和是否需补充验证。
2. 观点去重规则：同一高频观点只在一个核心章节集中展开，其他章节只补充新增证据。
3. 强化报告结构：报告调整为 0-9 章正式提案结构，包含一页核心结论、行业与入口变化、反方论证和品牌升级方向。
4. 视觉模型：强制包含品牌心智迁移图、用户场景链路图、竞品入口地图、品牌问题金字塔、品牌位置迁移图、12 个月品牌建设路线图。
5. 反方论证：主动挑战关键机会判断，防止报告只做单向论证。
6. HTML-first 生成：默认输出完整 HTML 报告，而不是纯 Markdown。
7. 提案级视觉系统：白背景、荧光高亮字/色块、黑色文字、极粗标题字重。
8. 写作风格指南：内置 `writing-guide.md` 提供逐章写作指导、常见错误警示和"好"的信号标准。
9. 质量自检清单：内置 13 项 LLM 自检清单，确保输出前完成质量验证。
10. 报告深度三档：支持简版/标准版/深度版，根据使用场景自动调整输出篇幅和章节覆盖。
11. 边缘场景适配：覆盖新创品牌、B2B 品牌、个人 IP 品牌、多业务线品牌、未上线品牌、有负资产品牌等特殊场景。
12. 完整 HTML 报告模板：`report-html-template.html` 覆盖 0-9 章全部小节，支持 neon 视觉系统一键生成。

## 5. HTML 视觉系统

新版报告默认使用 `report-html-template.html` 和 `styles/html-report.css`。

视觉约束：

- 核心视觉：白背景 + 荧光高亮字/色块 + 黑色文字
- 荧光色：`#C8FF00`
- 白色：`#FFFFFF`
- 黑色：`#1A1A1A`
- 中文字体：`PingFang SC`
- 英文字体：`Inter / Helvetica Neue`
- 标题：Inter 系无衬线 + 极粗标题字重
- 表格：黑色描边，荧光色表头
- 核心结论：使用荧光色块、荧光标签或黑色描边卡片突出

## 6. 依据类型

每个核心判断或结论后，必须标注以下依据类型之一或多个：

- 公开数据
- 竞品观察
- 用户调研
- 业务数据
- 专家判断
- 待验证假设

如果没有明确数据支持，必须标注为“待验证假设”，并说明需要通过用户调研、业务数据或竞品数据进一步验证。

## 7. 新版报告结构

最终报告标题：

```text
《{{品牌名称}}品牌诊断报告》
```

完整结构：

0. 一页核心结论
1. 品牌当前处境
2. 行业与入口变化
3. 竞品心智格局
4. 用户心智与场景洞察
5. {{品牌名称}}品牌资产与断点
6. 核心矛盾与战略机会
7. 品牌升级方向
8. 下一步验证与落地建议
9. 附录

## 8. 如何使用

1. 将品牌基础信息、行业信息、竞品信息、用户信息、品牌自身信息填入 `prompt-template.md`。
2. 按 `SKILL.md` 的强制规则生成完整 HTML 报告。当前项目所在文件系统对文件名大小写不敏感，`skill.md` 与 `SKILL.md` 指向同一份 Skill 主体文件。
3. 使用 `report-html-template.html` 作为 HTML 视觉模板。
4. 使用 `styles/html-report.css` 作为默认视觉样式。
5. 使用 `output-schema.md` 检查输出结构是否完整。
6. 需要 PDF 时，参考 `scripts/export-pdf.md`，优先通过 HTML 打印或 Playwright/Puppeteer 导出。
7. 如需章节级写作指导，参考 `writing-guide.md`。
8. 如需验证生成的 HTML 报告质量，参考 `scripts/validate-html.md`。

## 9. PDF 导出说明

报告默认先生成 HTML，再通过浏览器打印、Playwright 或 Puppeteer 导出 PDF。

PDF 样式要求：

- 字体：中文优先使用 PingFang SC，英文优先使用 Inter / Helvetica Neue
- 页面：A4
- 视觉：白背景、荧光高亮、黑色文字
- 一页核心结论和诊断依据页建议单独成页
- Mermaid 图示保留为代码块，方便支持 Mermaid 的环境渲染
- 表格避免过宽，宽表可拆分
- 12 个月路线图适合横向表格展示
- 风格简洁、专业、清晰、结构化

## 10. 注意事项

- 不要写死任何具体品牌。
- 不要在 Skill 主体中加入固定品牌分析内容。
- 不要输出简历相关内容。
- 不要编造具体数据。
- 不要把专家判断写成事实。
- 不要缩减为摘要报告。
- 报告深度默认为"深度版"，如需简版或标准版请在输入"报告深度"字段指定。
- 不要让第 2 章变成 PEST 分析——必须聚焦竞争逻辑变化。
- 不要让反方论证走过场——至少 1 条质疑需"部分成立"或"成立"。
