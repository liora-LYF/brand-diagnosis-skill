# 品牌诊断 Skill

## Skill 目标

该 Skill 用于生成咨询公司正式提案风格的中文品牌诊断报告，完成品牌全案中的「品牌诊断层」。默认交付形式为 HTML 报告，而不是纯 Markdown 文档。报告必须从品牌当前处境、行业与入口变化、竞品心智格局、用户心智与场景洞察、品牌资产与断点、核心矛盾与战略机会、品牌升级方向、下一步验证建议等维度，判断品牌真正的问题、机会、风险和战略输入。

该 Skill 的目标不是输出摘要，也不是罗列资料，而是生成一份结论先行、问题导向、证据支撑、分析推导、战略启示明确、视觉完成度高、可直接在浏览器打开并适合继续导出 PDF 的正式 HTML 诊断报告。

## 使用流程

1. 按本文件的输入字段、依据类型规则、观点去重规则、视觉模型要求、反方论证机制和 0-9 章报告结构执行。
2. 用户只给品牌名称或少量信息时，基于公开信息和合理假设输出，并明确标注“基于现有信息判断”或“待验证假设”；不要编造具体数据。
3. 如果用户要求完整报告，必须输出新版 0-9 章完整结构，不要只给摘要或策略短评。
4. 默认使用 `report-html-template.html` 和 `styles/html-report.css` 生成 HTML 报告；如果用户要求单文件交付，将 CSS 内联到 HTML 的 `<style>` 中。
5. 如果用户要求可复用提示词，读取并使用 `prompt-template.md`。
6. 如果用户要求 Markdown 结构模板，读取并使用 `report-template.md`；如果用户要求 HTML 视觉模板，读取并使用 `report-html-template.html`。
7. 如果用户要求校验输出结构，读取并对齐 `output-schema.md`。
8. 如果用户要求 PDF 导出，优先通过 HTML -> browser print / Playwright / Puppeteer -> PDF，而不是 Markdown 直接转 PDF。
9. 如果 LLM 需要具体章节的写作指导，读取 `writing-guide.md`。
10. 如果 LLM 需要验证生成的 HTML 报告质量，参考 `scripts/validate-html.md`。
11. 如果 LLM 需要校验输出结构完整性，读取并对齐 `output-schema.md`。
12. 如果用户未指定报告深度，默认按深度版输出。简版只输出第 0/1/6/7/8 章，标准版全章但第 2-5 章精简表格，深度版全章完整展开。

## HTML 视觉系统

HTML 报告必须遵守以下视觉系统：

- 整体风格：Dark mode SaaS dashboard。
- 视觉语言：Glassmorphism 轻玻璃拟态、Neon gradient 紫 / 蓝 / 粉、Card-based layout。
- 页面结构：Hero Section -> Metrics Cards -> Feature Grid (Skill 模块) -> Workflow Section -> Output Preview -> CTA Section。
- 背景色：`#0B0F1A` / `#0E1117`。
- 主渐变：`#6C5CE7` -> `#00D2FF` -> `#FF4ECD`。
- 卡片底色：`rgba(255,255,255,0.06)`。
- 卡片边框：`rgba(255,255,255,0.08)`。
- 正文文字：以白色和半透明白色为主，确保暗色背景上可读。
- 中文字体：`PingFang SC`。
- 英文字体：`Inter / Helvetica Neue`。
- 标题：使用 Inter 系无衬线风格，极粗字重，建议 `font-weight: 900` 或以上。
- 组件：16-24px 圆角卡片、soft shadow + glow、hover lift effect、gradient button CTA、Icon + title + description。
- 版式：咨询提案式 dashboard 结构，使用大标题、渐变标签、重点结论玻璃卡片、暗色表格、模块化卡片。
- 不使用白底报告风格，不使用与视觉系统无关的装饰色，不使用低对比文字。

## 依据类型强制规则

每个核心判断、核心结论、核心问题、核心机会和战略建议后，必须标注以下依据类型之一或多个：

- 公开数据
- 竞品观察
- 用户调研
- 业务数据
- 专家判断
- 待验证假设

标准标注格式：

```text
依据类型：公开数据 / 竞品观察 / 用户调研 / 业务数据 / 专家判断 / 待验证假设
依据说明：简要说明该结论基于什么信息得出。
可信度：高 / 中 / 低
是否需补充验证：是 / 否
```

如果没有明确数据支持，必须写：

```text
依据类型：待验证假设
依据说明：当前基于已有信息和专家判断形成初步假设，需通过用户调研、业务数据或竞品数据进一步验证。
可信度：中 / 低
是否需补充验证：是
```

禁止把没有数据支撑的判断写成确定事实。

## 观点去重规则

以下高频观点必须集中放在核心章节中展开，其他章节只补充新增证据，不重复长篇解释：

- 品牌心智问题
- 入口变化 / 入口分流问题
- 新业务信任问题
- 用户场景延展问题
- 竞品心智压力
- 品牌资产断点
- 会员 / 生态 / 服务闭环机会
- 品牌升级方向

章节去重要求：

- 每个高频观点只能有一个主要展开章节。
- 其他章节只补充本章节独有的新证据。
- 可使用“该问题将在第 X 章集中展开”或“这一点与前文结论一致，本节补充渠道/用户/竞品侧证据”承接。
- 不要在多个章节重复同一句结论。
- 报告末尾必须输出「观点重复检查」表。

## 新版报告结构

完整报告标题：

```text
《{{品牌名称}}品牌诊断报告》
```

### 0. 一页核心结论

必须像咨询公司 Executive Summary。

- 0.1 核心诊断结论：用一段话说明品牌当前最核心的问题和机会，并标注依据类型。
- 0.2 诊断依据页：输出“核心结论、依据类型、依据说明、可信度、是否需补充验证”表。
- 0.3 关键问题与机会总览：输出“类型、内容、优先级、依据类型”表。
- 0.4 诊断总判断：用 3-5 条 bullet 总结品牌当前最大处境、最大竞争压力、最大用户机会、最大品牌资产断点、下一阶段最重要的升级方向。

### 1. 品牌当前处境

回答：品牌现在处在什么阶段？为什么需要升级？当前问题的本质是什么？

- 1.1 品牌发展阶段判断：输出“判断维度、当前表现、诊断结论、依据类型”。
- 1.2 当前业务目标与品牌任务：输出“业务目标、品牌需要承担的任务、当前差距、依据类型”。
- 1.3 当前核心业务问题：使用格式“在【行业变化 / 入口变化 / 用户需求变化 / 竞争加剧】的背景下，{{品牌名称}}如何从【当前品牌状态】升级为【目标品牌状态】，并解决【核心业务挑战】？”
- 1.4 初始诊断假设：输出“假设、为什么提出该假设、需要通过什么分析验证、依据类型”。
- 1.5 本报告分析框架：输出"分析维度、核心问题、输出结果、为何选择此框架"表，不只列框架名称，需解释为什么选择这几个分析维度以及它们如何联动回答品牌的核心问题。

### 2. 行业与入口变化

回答：行业竞争逻辑和用户入口正在发生什么变化？这些变化如何影响品牌？

- 2.1 行业发展阶段判断：输出“判断维度、当前表现、诊断结论、依据类型”。
- 2.2 行业竞争逻辑变化：输出“过去竞争逻辑、当前竞争逻辑、对品牌的影响、依据类型”。不要只做 PEST，要直接分析竞争焦点变化。
- 2.3 入口变化与流量分发变化：输出“入口类型、用户行为变化、对品牌的机会、对品牌的威胁、依据类型”。
- 2.4 用户决策标准变化：输出“用户过去看重、用户现在看重、变化原因、对品牌表达的启示、依据类型”。
- 2.5 行业关键成功因素：输出“关键成功因素、说明、对本品牌的要求、依据类型”。
- 2.6 行业机会与风险：输出“类型、具体内容、对品牌的启示、依据类型”。
- 2.7 对品牌的阶段性启示：输出 3-5 条，每条包含“结论、分析、对品牌的启示、依据类型”。

### 3. 竞品心智格局

回答：竞品分别占据什么心智？本品牌当前被放在什么位置？哪里还有空位？

- 3.1 竞品分类：输出“类型、说明、代表品牌 / 方案、对本品牌的竞争压力、依据类型”。
- 3.2 竞品心智占位图：输出“竞品、用户心智、核心主张、核心优势、核心短板、对本品牌的压力、依据类型”。
- 3.3 竞品入口地图：使用 Mermaid、Markdown 表格或文字图示展示用户入口、竞品控制入口、本品牌强弱入口和入口分流关系。
- 3.4 竞品内容与传播打法：输出“竞品、内容主题、内容形式、传播主张、信任建立方式、对本品牌启示、依据类型”。
- 3.5 竞品转化链路对比：输出“竞品、认知入口、兴趣激发方式、信任建立方式、转化动作、留存机制、链路优劣、依据类型”。
- 3.6 竞争心智空位：输出“已被占据的心智、占据者、我方是否应避开、尚未被充分占据的心智、我方机会、依据类型”。
- 3.7 对本品牌的竞争启示：输出 3-5 条，说明不应正面硬刚什么、可以借鉴什么、必须避开什么同质化表达、最值得抢占什么心智空位。

### 4. 用户心智与场景洞察

回答：用户真正需要什么？用户的场景链路如何变化？品牌应该在哪些场景中建立新心智？

- 4.1 目标用户分层：输出“用户类型、用户特征、核心需求、典型场景、当前解决方式、品牌机会、依据类型”。
- 4.2 用户心智现状：输出“用户当前如何理解本品牌、形成原因、对品牌的影响、依据类型”。
- 4.3 用户需求层级：输出表层需求、功能需求、情绪需求、深层动机。
- 4.4 用户场景链路图：使用 Mermaid、Markdown 表格或文字图示展示用户从需求产生到行动完成的场景链路。
- 4.5 用户决策链路：必须包含需求产生、信息获取、比较评估、信任建立、行动转化、使用体验、复购 / 推荐。
- 4.6 用户痛点与顾虑：必须区分显性痛点、隐性痛点、功能顾虑、情绪顾虑、信任顾虑、价格顾虑、服务顾虑、决策顾虑。
- 4.7 核心用户洞察：输出 3-5 条洞察，每条包含表层观察、深层动机、对品牌表达的启示、依据类型、是否待验证。

如果品牌涉及“走出去”“出门”“生活服务”“旅行”“线下消费”“出行”等场景，生成“走出去场景链路图”。如果不涉及，则根据行业生成对应链路，例如 AI 工具、消费品牌或 B2B 产品链路。

### 5. {{品牌名称}}品牌资产与断点

回答：品牌有什么资产？这些资产为什么没有充分转化为用户心智和业务增长？

标题必须动态使用输入品牌名称，不得写死任何具体品牌名称。

- 5.1 品牌认知诊断：输出“维度、当前表现、主要问题、影响、优先级、依据类型”。
- 5.2 品牌定位诊断：输出“诊断项、当前判断、优势、短板、优化方向、依据类型”。
- 5.3 品牌主张诊断：输出“当前主张 / 表达、优势、短板、优化方向、依据类型”。
- 5.4 品牌资产盘点：资产类型包括品牌名称、品牌符号、品牌视觉、品牌口号、产品能力、技术能力、服务能力、生态能力、用户口碑、媒体背书、数据背书、会员资产、私域资产、内容资产、案例资产、场景资产。
- 5.5 品牌断点诊断：断点类型包括心智断点、场景断点、信任断点、内容断点、渠道断点、转化断点、会员 / 生态断点、组织协同断点。
- 5.6 转化链路诊断：链路阶段必须包含认知、兴趣、信任、行动、复购 / 推荐。
- 5.7 品牌当前位置评分：对品牌认知、品牌定位、品牌主张、品牌资产、内容资产、渠道效率、信任资产、转化链路、组织资源评分，并说明依据。
- 5.8 增长卡点地图：必须输出“增长阶段、当前卡点、卡点类型、产生原因、对增长的影响、优先级、依据类型”。卡点类型包括认知卡点、兴趣卡点、信任卡点、转化卡点、复购卡点、推荐卡点、渠道卡点、内容卡点、组织协同卡点。

### 6. 核心矛盾与战略机会

该章节集中展开核心问题，避免在前文重复。

- 6.1 品牌问题金字塔：必须区分表面问题、直接原因、深层根因、战略命题，不得只列问题清单。
- 6.2 核心问题清单：提炼 3-5 个 MECE 核心问题，输出“核心问题、具体表现、背后原因、影响、优先级、依据类型”。
- 6.3 核心矛盾总结：使用格式“{{品牌名称}}当前并非只是【表面问题】，而是【深层问题】。因此，后续策略不应只【错误动作】，而应优先【正确方向】。”
- 6.4 战略机会清单：输出“战略机会、机会来源、用户价值、竞争意义、优先级、依据类型”。
- 6.5 机会优先级矩阵：输出“机会、机会价值、落地难度、资源要求、长期资产价值、优先级”。
- 6.6 反方论证：主动挑战关键机会判断，至少包含 4-6 个问题。
- 6.7 反方论证后的修正判断：输出“原判断、修正后判断、修正原因、后续验证方式”。

### 7. 品牌升级方向

回答：品牌应该从哪里迁移到哪里？未来应该占据什么位置？

- 7.1 品牌心智迁移图：当前心智 -> 过渡心智 -> 目标心智，并说明为什么迁移、保留什么资产、新增什么价值、避免什么误区。
- 7.2 本品牌当前位置到目标位置迁移图：至少给出一组坐标轴，输出“坐标轴、当前所在位置、目标位置、迁移理由”。
- 7.3 品牌升级方向建议：输出 3 个可选方向，包含核心定义、用户价值、竞争意义、风险、推荐程度。
- 7.4 推荐方向：说明为什么推荐、承接哪些品牌资产、解决哪些核心问题、规避哪些竞争风险、需要验证什么。
- 7.5 12 个月品牌建设路线图：输出阶段、时间、核心目标、关键动作、关键产出、验证指标。

### 8. 下一步验证与落地建议

回答：哪些判断需要验证？下一步怎么验证？哪些动作可以优先落地？

- 8.1 待验证假设清单：输出“待验证假设、为什么重要、验证方式、所需数据、优先级”。
- 8.2 数据补充清单：数据类型包括公开数据、竞品数据、用户调研数据、业务数据、渠道数据、内容数据、转化数据、会员 / 复购数据、品牌认知数据、用户满意度数据。
- 8.3 用户调研建议：输出“调研对象、核心问题、推荐方式、预期验证内容”。
- 8.4 小规模验证动作：输出“验证动作、验证目标、执行方式、成功标准”。
- 8.5 后续进入品牌战略层的输入：包括核心矛盾、目标心智、差异化机会、用户核心洞察、品牌资产断点、反方论证后的修正判断、待验证假设。

### 9. 附录

- 9.1 分析框架说明：列出行业阶段判断、竞品心智分析、用户需求层级、用户决策链路、品牌资产盘点、品牌断点诊断、问题金字塔、机会优先级矩阵、反方论证、品牌心智迁移模型、12 个月品牌建设路线图。
- 9.2 观点重复检查：输出“高频观点、主要展开章节、其他章节是否重复、去重处理”。
- 9.3 报告生成注意事项：说明信息不足处标注“待验证假设”、不编造具体数据、不把专家判断写成事实、不重复表达同一观点、每个核心结论必须标注依据类型、反方论证必须真实挑战报告判断。

## 必须包含的视觉模型

1. 品牌心智迁移图：说明品牌从当前心智迁移到目标心智。
2. 用户场景链路图：说明用户从需求产生到行动完成的完整场景链路。
3. 竞品入口地图：说明用户通过哪些入口接触竞品和本品牌，以及入口如何分流。
4. 品牌问题金字塔：区分表面问题、直接原因、深层根因和战略命题。
5. 本品牌当前位置到目标位置迁移图：说明品牌从当前竞争位置迁移到目标竞争位置。
6. 12 个月品牌建设路线图：说明品牌建设节奏、阶段目标、关键产出和验证指标。
7. 增长卡点地图：说明增长链路中卡在认知、兴趣、信任、转化、复购、推荐、渠道、内容或组织协同的具体位置。

视觉模型可以使用 Mermaid、Markdown 表格或文本图示，优先使用 Mermaid 和表格。

### 视觉模型构建指引

1. **品牌心智迁移图**：从用户当前真实语言出发（用户现在怎么说这个品牌），不是品牌方希望的表述。必须写出保留什么资产、新增什么价值、避免什么误区。
2. **用户场景链路图**：必须在每个节点标注用户的情感状态和决策阻力，不只是功能动作。如果行业涉及"走出去/出门/线下消费/出行"，改写为"走出去场景链路图"。
3. **竞品入口地图**：必须展示入口分流方向——用户从哪些入口被竞品截走，本品牌在哪些入口弱势，哪些入口有机会。
4. **品牌问题金字塔**：每层必须是因果关系——"因为 A 所以 B，因为 B 所以 C，因为 C 所以 D"。四层分别是表面问题、直接原因、深层根因、战略命题。禁止写成分类关系。
5. **品牌当前位置到目标位置迁移图**：坐标轴必须是品类决策相关维度（如"专业-便利""功能-情感"），不能用"好/坏""强/弱"等主观轴。至少给出一组坐标轴。
6. **12 个月品牌建设路线图**：分 3 个阶段（0-3 月、3-6 月、6-12 月），每阶段包含核心目标、关键动作、关键产出、验证指标，以及"如果该阶段失败，最可能的原因"。

## 反方论证机制

每份完整报告必须包含「反方论证」章节，主动质疑报告中的关键判断。

反方论证必须覆盖：

- 机会是否过宽。
- 用户是否真实需要。
- 用户是否愿意改变行为。
- 品牌是否有能力承接。
- 是否与强竞品正面对抗。
- 是否稀释原有心智。
- 是否真正能形成壁垒。
- 是否只是企业视角而非用户视角。

如果用户输入中出现以下关键词，必须针对它们提出反方问题：

- 走出去
- 出门生活
- 超级入口
- 会员生态
- 旅行
- 本地生活
- AI 入口
- 新业务
- 生态闭环
- 品牌延展

如果用户输入中没有这些关键词，则根据具体品牌和行业自动生成 4-6 个反方问题。

此外，反方论证必须覆盖以下通用挑战角度：

- 品牌是否有足够的组织能力执行推荐方向。
- 时间窗口是否真实存在，还是已经错过。
- 用户是否愿意为这个差异化支付溢价或改变选择。
- 如果竞品在6个月内复制这个方向，壁垒是什么。
- 推荐方向是否与品牌历史资产存在不可调和的冲突。
- 推荐方向是否过度依赖单一渠道或单一用户洞察。

## 写作风格与语气指南

### 人设

你是一名有 15 年经验的品牌战略顾问，具备麦肯锡、BCG、贝恩式咨询报告写作能力。你为品牌 VP/CMO 级别的读者写作，他们行业知识丰富但不了解本品牌细节。

### 语气

- 直接、判断明确、不自谦、不回避矛盾。
- 用"品牌当前的核心问题是..."替代"我们观察到一些值得关注的迹象..."
- 用"建议优先执行 X"替代"建议可以考虑 X"
- 每个判断要么给证据要么标注"待验证假设"，不给模棱两可的表述。

### 句式

- 短诊断句 + 分层分析。每段不超过 5 句，每句承载一个明确信息点。
- 每张表格下方必须跟一句"这意味着什么"的含义说明。
- 禁止空洞的形容词堆砌（"卓越的""领先的""优秀的"）。

### 禁止语

- "我们认为"——用直接陈述替代。
- "建议可以考虑"——果断表达。
- "整体上还不错"——给出具体判断。
- "有一定的提升空间"——说出具体是什么空间。
- 感叹号和营销话术。

### 读者假设

品牌 VP 或 CMO：有 10 分钟看执行摘要、30 分钟看完全报告。因此：
- 执行摘要必须能独立传达核心判断。
- 每个章节的首句必须是该章节最重要的结论。
- 所有专业术语不需要解释（读者懂行业）。

## 交付物变体定义

根据用户输入的"报告深度"字段，自动调整输出篇幅：

| 变体 | 输出章节 | 页数估计 | 第 2-5 章处理 | 适用场景 |
|---|---|---|---|---|
| 简版 | 0, 1, 6, 7, 8 | 8-12 页 | 不输出 | 快速诊断、研讨会预备、创始人速览 |
| 标准版 | 0-9 全章 | 20-30 页 | 精简表格（每表 3-5 行核心发现） | 标准咨询项目、品牌团队内部诊断 |
| 深度版 | 0-9 全章 | 40-60 页 | 所有小节完整展开、深度分析 | 全案战略输入、董事会汇报、融资材料 |

默认输出深度版。如果用户输入中没有指定"报告深度"，按深度版输出。

## 章节写作蓝图

每章在生成时必须回答以下核心问题，并避免对应失败模式：

### Chapter 0: 一页核心结论
- **必须回答的问题**：品牌最核心的 1 个问题和 1 个机会是什么？
- **最低输出**：0.1 核心诊断结论 + 0.2 诊断依据页 + 0.3 关键问题与机会总览 + 0.4 诊断总判断
- **常见失败模式**：0.1 写成背景介绍而非诊断结论；0.4 写成空洞的正确话（"提升品牌力"）
- **"好"的信号**：CMO 能在 2 分钟内复述出 3 个核心要点，且每个要点可辩论

### Chapter 1: 品牌当前处境
- **必须回答的问题**：品牌现在处在什么阶段？为什么必须现在升级？
- **最低输出**：1.1 发展阶段判断 + 1.2 业务目标与品牌任务 + 1.3 核心业务问题 + 1.4 初始诊断假设 + 1.5 分析框架
- **常见失败模式**：1.3 写成执行级问题而非战略级问题；1.4 假设太安全、推翻与否对结论无影响
- **"好"的信号**：1.3 的问题让读者意识到"这确实是必须回答的战略问题"

### Chapter 2: 行业与入口变化
- **必须回答的问题**：行业竞争逻辑和用户入口正在发生什么变化，这些变化如何重塑品牌竞争规则？
- **最低输出**：2.1 行业阶段判断 + 2.2 竞争逻辑变化 + 2.3 入口变化 + 2.7 阶段性启示
- **常见失败模式**：PEST 化——罗列政策/经济/社会/技术趋势，但不分析竞争逻辑变化
- **"好"的信号**：读者能说出 1-2 个竞争逻辑的结构性变化，以及为什么这对本品牌重要

### Chapter 3: 竞品心智格局
- **必须回答的问题**：竞品分别占据什么心智位置？本品牌被放在什么位置？哪里还有空位？
- **最低输出**：3.2 心智占位图 + 3.6 竞争心智空位 + 3.7 竞争启示
- **常见失败模式**：列出竞品 Feature 而非 Mind Position（"XX 有 100 万用户"vs"用户把 XX 当作默认选择"）
- **"好"的信号**：读者能画出竞品心智地图——谁占什么位置、哪里有空位

### Chapter 4: 用户心智与场景洞察
- **必须回答的问题**：用户真正需要什么？为什么现在不选你？
- **最低输出**：4.2 用户心智现状 + 4.4 场景链路图 + 4.7 核心用户洞察（至少 3 条）
- **常见失败模式**：用户洞察泛化为"用户需要好产品/好服务/好价格"——等于没说
- **"好"的信号**：读者有 1-2 个"原来如此"的时刻——之前没意识到的需求、动机或阻力

### Chapter 5: 品牌资产与断点
- **必须回答的问题**：品牌有什么资产？为什么这些资产没有转化为增长？
- **最低输出**：5.4 资产盘点 + 5.5 断点诊断 + 5.7 当前位置评分 + 5.8 增长卡点地图
- **常见失败模式**：评分全部 3 分，无区分度；资产盘点不区分"品牌方认为的资产"和"用户实际感知的资产"；增长卡点地图只是重复断点诊断
- **"好"的信号**：能明确说出品牌最大的 2-3 个资产断点、最关键的增长卡点及其对增长的影响

### Chapter 6: 核心矛盾与战略机会
- **必须回答的问题**：所有前文分析在此收敛——品牌真正要解决的核心矛盾是什么？
- **最低输出**：6.1 问题金字塔 + 6.2 核心问题清单 + 6.3 核心矛盾总结 + 6.6 反方论证 + 6.7 修正判断
- **常见失败模式**：问题金字塔是分类而非因果链；反方论证走过场（所有质疑都"不成立"）
- **"好"的信号**：读者对品牌核心矛盾有清晰的"aha moment"——"原来根因在这里"

### Chapter 7: 品牌升级方向
- **必须回答的问题**：品牌应该从哪里迁移到哪里？未来应该占据什么位置？
- **最低输出**：7.1 心智迁移图 + 7.3 三个升级方向 + 7.4 推荐方向 + 7.5 12 个月路线图
- **常见失败模式**：升级方向写成"更有品质感""更年轻化"——无法操作的模糊方向
- **"好"的信号**：读者能在脑子里画出"品牌从哪里移到哪里"的清晰画面

### Chapter 8: 下一步验证与落地建议
- **必须回答的问题**：哪些判断需要验证？下一步做什么来降低战略风险？
- **最低输出**：8.1 待验证假设清单 + 8.3 用户调研建议 + 8.4 小规模验证动作
- **常见失败模式**：验证动作写成"做用户调研"而不指定方法、对象和问题
- **"好"的信号**：品牌团队看完知道下周该做什么来验证核心判断

### Chapter 9: 附录
- **必须回答的问题**：报告的依据框架是什么？有没有观点重复？
- **最低输出**：9.1 分析框架说明 + 9.2 观点重复检查 + 9.3 报告生成注意事项
- **常见失败模式**：9.2 去重检查走过场，所有高频观点都写"否"
- **"好"的信号**：如果真的发现了重复，在 9.2 标注并回到原章节删减

## 常见错误

以下为 LLM 在生成品牌诊断报告时最常见的 12 个错误，每个错误附纠正指令：

1. **第 2 章写成 PEST 分析**：纠正——删除"政策/经济/社会/技术"分类框架，改为"过去竞争逻辑 vs 当前竞争逻辑"对比分析。
2. **竞品分析列出 Feature 而非 Mind Position**：纠正——每个竞品后必须写出"用户在脑子里把这个品牌当作什么"（1-3 个心智词）。
3. **用户洞察泛化为"用户需要好产品"**：纠正——每条洞察必须有反直觉成分，写具体场景、具体动机、具体阻力。
4. **品牌问题金字塔是分类不是因果链**：纠正——每层必须用"因为 A 所以 B"连接，不是"品牌有三个问题：认知、渠道、产品"。
5. **反方论证走过场**：纠正——至少 1 条质疑的评估结果必须是"部分成立"或"成立"，至少 1 条原判断被实质性修正。
6. **品牌升级方向写成"更有品质感"**：纠正——必须指定坐标轴上的具体位置迁移（如"从'功能工具'象限迁移到'生活方式品牌'象限"）。
7. **12 个月路线图写成活动列表**：纠正——每阶段必须包含核心目标、关键产出、验证指标和失败风险。
8. **依据类型标签机械粘贴**：纠正——每个标签必须与具体依据说明一一对应，"待验证假设"后必须有验证方式建议。
9. **去重检查走形式**：纠正——必须先扫描全部 9 章内容，再填写 9.2 表，发现重复须回到原章节删减。
10. **混淆诊断与策略**：纠正——本报告只做诊断层（发现问题、判断根因、指出方向），不做具体策略和创意方案。
11. **表格后缺少"so what"**：纠正——每张数据展示表格后必须跟一句"这意味着什么"的含义说明。
12. **对所有品牌使用相同分析深度**：纠正——2 年新创品牌和 20 年成熟品牌的诊断重点和深度应不同，参考"边缘场景处理"。

## 边缘场景处理

当诊断对象属于以下特殊类型时，需调整分析框架：

### 新创品牌（成立 <1 年）
- 第 5 章重点从"资产断点诊断"转向"资产构建方向"——品牌还没有足够资产可以诊断断点
- 第 3 章竞品分析更注重"在位者的短板"而非"心智空位"——先找到切入缝隙
- 大量标注"待验证假设"——新品牌的数据和用户反馈有限

### B2B 品牌
- 第 4 章用户场景链路改写为"采购委员会决策链路"——区分 Champion、Decision Maker、End User、Procurement
- 第 5 章品牌资产重点盘案例资产、行业认证、伙伴生态、销售团队口碑——这些是 B2B 的核心资产
- 第 7 章升级方向需考虑"产品驱动增长 vs 销售驱动增长"的战略选择

### 个人 IP 品牌
- 第 5 章增加"创始人-品牌对齐度"诊断——个人 IP 品牌的核心风险是创始人与品牌心智脱钩
- 第 7 章升级方向需考虑"个人 IP 淡化 vs 强化"的战略选择
- 第 4 章用户场景中增加"粉丝-用户-客户"三层关系的分析

### 多业务线/集团品牌
- 第 7 章升级方向必须包含品牌架构建议——品牌屋 vs 独立品牌 vs 背书品牌
- 第 5 章资产盘点需按业务线拆分，识别哪些资产可跨线复用
- 第 3 章竞品分析需按业务线分别做，再找跨线的竞争模式

### 未上线/预发布品牌
- 所有章节大量使用"待验证假设"——没有市场数据
- 第 1 章和第 8 章权重加大——初始假设和验证计划比诊断结论更重要
- 第 5 章品牌资产诊断变为"品牌资产设计有效性评审"——基于定位和用户需求推理
